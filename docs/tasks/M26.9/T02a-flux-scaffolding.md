# T02a: Flux root scaffolding for staging

> **Milestone**: M26.9-staging-cluster-and-post-merge-e2e
> **Status**: Not started
> **Estimate**: S (2-3h)
> **Depends on**: T01 (staging cluster terraform merged; apply can happen in
> parallel since T02a lands YAML only)

---

## Description

Create a new Flux root at
`infrastructure/kubernetes/flux/clusters/staging/` that mirrors the
prod root layout. **No workload patches in this task** — patches land
in T02b/c/d. T02a delivers the Flux dependency chain (flux-system →
namespaces → operators → data → services → apps) and the SOPS age-key
secret scaffolding so later tasks can layer on top without churning the
directory shape.

## Subtasks

- [ ] **Flux system objects** at
  `infrastructure/kubernetes/flux/clusters/staging/flux-system/` rendered
  from `flux bootstrap github --dry-run`. Counts as generated per the
  AGENTS.md small-PR exception.
- [ ] **Root kustomization chain** mirroring prod's dependency graph
  (namespaces → operators → data → services → apps), with the
  reconcile interval dropped to 1m for staging.
- [ ] **Namespaces** for `platform`, `monitoring`, any others prod uses.
- [ ] **Operators** (CNPG operator, kube-prometheus-stack operator, KEDA
  if prod has it) as HelmRelease references — installed via the base,
  not re-declared in staging.
- [ ] **SOPS age-key secret machinery**:
  `infrastructure/kubernetes/flux/clusters/staging/secrets/` holds the
  encrypted secret(s) staging needs to bootstrap. Committed encrypted.
  The age keypair is generated by the operator during
  `flux bootstrap`; its private half lives in the staging GitHub
  Actions runner secret set (T03's concern for E2E runs).
- [ ] **No** overlay patches, no CNPG cluster manifests, no Prometheus
  HelmRelease tuning, no ingress hostNetwork patch in this task.

## Implementation Notes

1. **Reconcile interval 1m** is the staging-specific default (prod is
   5m). Faster feedback; single node can handle the load.
2. **Empty data/services entries are OK.** The Flux kustomizations need
   to exist even if they resolve to empty overlay paths. This keeps
   the dependency chain intact while T02b-d fill each layer in later
   PRs. Kustomize accepts an empty resources list.
3. **Overlay path for apps** points at
   `infrastructure/kubernetes/overlays/hetzner/staging/`. That
   directory does NOT exist at T02a merge time; T02b creates it. Flux
   reconciliation will fail until T02b lands — acceptable since
   terraform apply also hasn't happened.
4. **SOPS age key provisioning** requires operator action: generate a
   new age keypair, encrypt the staging secret(s) to both that public
   key and the prod key (so break-glass works), commit encrypted, stash
   the private half in 1Password and the GitHub environment secret
   used by staging workflows.
5. **No behavior changes** to prod Flux or prod overlays in this task.
   The split milestone design decisions pin the prod path untouched.

## Acceptance Criteria

- [ ] `kubectl kustomize infrastructure/kubernetes/flux/clusters/staging/`
  renders without error.
- [ ] `flux --context staging-dry-run validate` (using kustomize output
  piped to `kubectl apply --dry-run=client`) succeeds for every
  resource.
- [ ] `sops --decrypt` round-trips the staging secret against the new
  age key.
- [ ] No files touched under `infrastructure/kubernetes/flux/clusters/production/`
  or `infrastructure/kubernetes/overlays/hetzner/production/`.
- [ ] PR diff stays inside the small-PR budget (under roughly 400
  hand-written LOC and 8 touched non-generated files).
  `gotk-components.yaml` counts as generated per the AGENTS.md rule.

## Verification

```bash
# Kustomize render
kubectl kustomize infrastructure/kubernetes/flux/clusters/staging/ > /tmp/staging-render.yaml
# Client-side validation
kubectl apply --dry-run=client -f /tmp/staging-render.yaml

# SOPS round-trip
sops --decrypt infrastructure/kubernetes/flux/clusters/staging/secrets/runtime-secrets.sops.yaml \
  | diff - <(sops --decrypt infrastructure/kubernetes/flux/clusters/staging/secrets/runtime-secrets.sops.yaml)
```

`flux bootstrap ... --path=infrastructure/kubernetes/flux/clusters/staging`
is the operator action that lands this against a real staging cluster.
That step is out of scope for the PR gate; it runs once T01's terraform
apply puts a kube API at the staging floating IP.
