## `Path(__file__).resolve().parents[N]` ascent depths

Almost every script under `tools/scripts/` uses `parents[2]` to ascend to
repo root (one level up from `tools/scripts/`, then one more to repo root).
After Phase 4 moves scripts into `tools/scripts/<subdir>/`, every one of
these must change to `parents[3]`. The `tests/architecture/test_repo_root_ascent_fallback.py`
test currently pins shell-script ascent depths; Phase 4 extends it to cover
Python.

**Action item for Phase 4:** before any move, grep `Path(__file__).resolve().parents[`
across the moved set and rewrite the `[N]` literal in lockstep with the new
directory depth. The `tests/architecture/test_setup_sip.py` and
`tests/architecture/test_github_runner_inventory.py` tests both compute
`REPO_ROOT = Path(__file__).resolve().parents[2]` and then construct
`MODULE_PATH = REPO_ROOT / "tools/scripts/<name>.py"` — those constants
need updating in the same commit as the script move.

## Smoke test safety net (committed in Phase 1)

`tests/architecture/test_tools_scripts_smoke.py` provides two layers:

1. **`test_script_parses_as_python`** — `ast.parse()` every `tools/scripts/**/*.py`. Catches syntax errors and file corruption from `git mv`. 62 parametrized cases, runs in <100 ms.
2. **`test_script_module_loads`** — `importlib.util.spec_from_file_location` in an isolated subprocess for every script with an `if __name__ == "__main__":` guard. Catches broken imports (the M26.6 failure mode: `from api_inventory_lib import X` after the module moves) and `Path(__file__).resolve().parents[N]` ascent bugs at module level. 58 parametrized cases, runs in ~10 s.

Both layers self-discover via `rglob`, so files moved in Phase 4 get picked
up automatically without an allowlist. Verified green: **120 / 120 pass**
on `refactor/simplify-py-scripts` baseline.
