## Goal

Implement the Affidea-specific booking state, provider client boundary, and
tool aliases needed by the reused demo prompts. This milestone keeps live-turn
booking/auth calls in solution-owned tools and services, with explicit typed
state for front desk, consultation, MRI, auth, slot locking, and booking.

## Design Decisions

1. Affidea business behavior lives in `solutions/appointment_booking`; Grove
   remains product-agnostic.
2. The reused prompt names stay usable. Where the prompt calls
   `search_available_slots`, `select_slot`, `transfer_to_mri_booking`,
   `lookup_user_by_id`, or similar demo names, the solution provides explicit
   alias tools or adapters instead of editing the prompt text.
3. Provider API clients are async, constructor-injected, and isolated behind
   solution-owned services. No `requests`, no direct `os.environ` reads in
   business logic.
4. Slot locking and booking writes must be idempotent before any retry is
   allowed.
5. Authentication state and patient details are protected tool/service state;
   reusable prompt and KB files must never contain hidden patient data.
6. Per-turn slot search, lock, auth lookup, and booking calls do not become
   Temporal activities. Temporal remains the durable outer call lifecycle and
   post-call owner.

## Acceptance Criteria

- [ ] The solution has typed request/response models for provider data, slots,
      slot locks, bookings, patient lookup, and identity verification.
- [ ] Consultation happy path can search, select, confirm price, collect visit
      reason, and create a booking through tool/service calls.
- [ ] MRI happy path can collect exam type, machine type, questionnaire answers,
      weight, city/time preference, slot choice, price confirmation, and booking.
- [ ] MRI unsafe answers route to human handoff only after the required
      confirmation step described in the reused prompt.
- [ ] Auth tools enforce sequential lookup -> STT keyword load -> name verify
      behavior; tests prove lookup and keyword loading are not called together.
- [ ] Tool names used by the copied prompts are registered for the Affidea agent
      profile.
- [ ] Structured errors include bucket A/B/C semantics from the design doc and
      enough context for logs without leaking patient data.
