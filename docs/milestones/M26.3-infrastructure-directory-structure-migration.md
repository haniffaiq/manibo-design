## Goal

Replace the split `infra/` plus `clusters/` layout with one honest `infrastructure/` root that matches actual ownership: Terraform is isolated by provider, Kubernetes keeps one shared base plus explicit overlays for local default/offline, CI, Hetzner, and GCP targets, Flux cluster roots live under Kubernetes, SOPS policy lives under infrastructure for both Hetzner and GCP secret paths, and provider-specific database choices stop pretending to be shared.

## Current Status Snapshot

1. `infrastructure/kubernetes/flux/clusters/production` is the live Flux reconciliation root for production.
2. `infrastructure/kubernetes/base/**` plus `infrastructure/kubernetes/overlays/{local/default,local/offline,ci}` now own the shared local/CI Kubernetes bootstrap surface, and local/default plus CI each own their own CNPG manifests instead of inheriting them from the shared base.
3. `infrastructure/terraform/hetzner/environments/production` now owns the live production Terraform/support-stack surface, while `infrastructure/kubernetes/overlays/hetzner/production/**` owns the live production runtime overlay and Helm values.
4. `infrastructure/terraform/hetzner/shared/ci-runner` now owns the Hetzner-only runner stack instead of the old fake-neutral top-level directory.
5. `infrastructure/sops/.sops.yaml` now owns the live repo-wide SOPS policy/config contract, while Hetzner Vault transit bootstrap stays under `infrastructure/terraform/hetzner/shared/vault-sops`.
6. `infrastructure/terraform/gcp/nfq/environments/{production,staging}` now own the live NFQ GCP wrapper contract, `infrastructure/terraform/gcp/nfq/modules/shared-k8s` owns the shared manifest-path surface those wrappers still export, and Manibo-only GCP stacks live under `infrastructure/terraform/gcp/manibo/**`.
7. `infrastructure/scripts/hetzner/{production,shared}` now owns the environment-specific Hetzner operational entrypoints, while repo-wide CI and harness helpers intentionally stay in `tools/scripts/**`.
8. The remaining legacy `infra/environments/{local,ci,shared,tenants}` roots plus `infra/environments/hetzner/staging/**` were fully retired after their live dependencies moved under `infrastructure/**`.
9. Top-level `infra/` and `clusters/` are now gone; `infrastructure/` is the only live infrastructure root.
10. The live `manibo-production` Flux sync path cutover was captured in a proof YAML (since removed after merge) before `clusters/production` was removed.

Only create provider/environment directories, overlay directories, and Flux cluster roots when a real source tree or active consumer exists. The current repo evidence guarantees one live Flux cluster root today, one live Hetzner Terraform environment root today, and three actively consumed local/CI overlay variants today: `local/default`, `local/offline`, and `ci`. It does not yet prove a standalone GCP-specific Kubernetes overlay tree, so `overlays/gcp/**` is created only when concrete GCP Kubernetes artifacts or consumers appear.

## Design Decisions

1. **One `infrastructure/` root only.** Top-level `infra/` and `clusters/` are retired after migration; no permanent dual-path compatibility fiction.
2. **Terraform is provider-isolated.** `terraform/gcp/**` and `terraform/hetzner/**` own their own modules and environment roots. We do not keep a fake cross-cloud `terraform/modules/`.
3. **Global SOPS policy lives under infrastructure.** The repo-owned SOPS config and secret-path contract move under `infrastructure/sops/**` and cover every secret-bearing path moved by this milestone, including local/CI overlay secrets, active Flux cluster secrets, and provider-specific Hetzner/GCP secret paths. Provider bootstrap for a specific backend such as Hetzner Vault transit or a future GCP-specific backend stays under that provider's Terraform tree.
4. **Hetzner-only stacks stay under Hetzner.** `ci-runner` and `vault-sops` are provider-specific root stacks under `terraform/hetzner/shared/**`, not fake-neutral top-level stacks and not duplicated as reusable modules.
5. **Kubernetes keeps one shared `base/`.** `base/` holds only manifests genuinely shared across local, CI, Hetzner, and GCP targets.
6. **Provider and environment differences live in overlays.** `local/default`, `local/offline`, `ci`, and provider-owned `<active-environment>` overlays own their real divergence instead of hiding it in conditionals or duplicated bases.
7. **Database ownership is provider-specific.** Activated Hetzner overlays own CNPG manifests; GCP owns Cloud SQL in Terraform and only carries Kubernetes connectivity/runtime wiring in overlays.
8. **Flux cluster roots live under Kubernetes.** Real cluster reconciliation entrypoints move to `kubernetes/flux/clusters/*`, one directory per actual cluster, and future roots are created only when the cluster is real.
9. **Migration is atomic at the path-contract level.** Workflows, scripts, tests, docs, and release-image automation move with the tree. No half-migrated path aliases that rot forever.
10. **No speculative abstractions.** If GCP and Hetzner do not share a thing, they do not share a directory just because the names look similar.
11. **Script ownership must be explicit.** Repo-wide CI, harness, and validation tooling stays in `tools/scripts/**`; only environment-owned operational scripts move under `infrastructure/scripts/**` if they are truly infrastructure-owned.

- No GCP production rollout work beyond path-safe structure and contracts.
- No Terraform toolchain swap or new orchestration layer.
- No permanent compatibility symlink or duplicate tree just to avoid updating references.
