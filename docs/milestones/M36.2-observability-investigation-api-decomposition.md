# M36.2: Observability Investigation API Decomposition

Status: done
Created: 2026-04-14

Depends on: M36.1
Reference: `wiki/architecture/architecture.md`, `wiki/design-docs/fastapi-best-practices.md`, `docs/arch/generated/api_inventory.md`

## Goal

Reduce route-layer entropy in the observability investigation API by breaking
`apps/api/src/platform_api/routes/observability/router.py` into thin transport
modules and moving reusable diagnosis logic into
`packages/platform-core/src/platform_core/observability/`. This milestone is
behavior-preserving and diagnosis-first: it does not redesign the web UI, does
not migrate metrics dashboards, and does not bundle the KPI/report routes.

## Design Decisions

1. **Problem-finding comes first** - the purpose of the observability API is to
   help operators and admins find system problems quickly, not to replace
   Grafana or GCP Monitoring.
2. **Use the new API-shell boundary rule** - `platform_api` owns HTTP
   transport, while `platform_core` owns reusable platform logic. If code can
   run without FastAPI, it moves down.
3. **Investigation API only** - scope is
   `apps/api/src/platform_api/routes/observability/router.py` and its package
   surface. `reports.py` and KPI/dashboard cleanup are deferred to a later
   milestone.
4. **Behavior-preserving only** - endpoint paths, auth scopes, query
   parameters, response shapes, mounted router behavior, and API inventory stay
   unchanged.
5. **Thin route modules, explicit owners** - route files stay transport-shaped.
   Route-owned schemas, dependencies, and presenters live in explicit
   package-local files. Reusable read models and diagnosis logic live in
   `platform_core`.
6. **No generic route framework** - reduce duplication with explicit shared
   builders and presenters, not with meta-router abstractions.
7. **Keep imports stable before narrowing** - preserve the package-level
   `platform_api.routes.observability` surface first, then narrow private
   compatibility exports only when caller proof says it is safe.
8. **Delete lines, not just move lines** - the target is net code reduction
   across the observability route package, not a file shuffle.
9. **No checklist coupling** - `docs/requirements/checklist.md` is not part of
   this milestone contract.

## Acceptance Criteria

- [x] `apps/api/src/platform_api/routes/observability/router.py` is deleted or
      reduced to a thin compatibility/assembly module under 500 LOC.
- [x] Endpoint route modules and thin package shims under
      `apps/api/src/platform_api/routes/observability/**` stay below 500 LOC;
      over-target support modules are explicitly named as shrink-only debt
      instead of being silently blessed.
- [x] Inline Pydantic investigation response models no longer live in the main
      route file.
- [x] Workflow payload decoding and workflow-step parsing no longer live in the
      route layer.
- [x] Call, workflow, channel, interactive-session, control-plane, and tenant
      composition investigation builders are reusable and owned below the API
      shell.
- [x] `platform_api.routes.observability` no longer exports unnecessary private
      helpers as its long-term package surface.
- [x] `tools/scripts/check_api_inventory.py` passes without published API drift.
- [x] Route topology/import-surface/file-size guards fail if the investigation
      API regrows into the previous god-module shape.
- [x] `apps/api/src/platform_api/routes/observability/reports.py` remains
      explicitly deferred to a later KPI/report milestone.
