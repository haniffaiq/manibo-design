## Goal

Finish the remaining production infrastructure work required to make the NFQ GCP
runtime usable from the public internet with a dedicated NFQ auth surface that
does not reuse the live Manibo public login contract.

This milestone closes the gap between "private runtime is healthy" and
"operators can log in through a public NFQ URL":

- stand up the public GCP edge for both web and API
- create temporary public DNS under `jakitlabs.com`
- configure a dedicated NFQ OIDC client/provider path, separate from Manibo
- verify database state, migrations, runtime health, and end-to-end login

## Design Decisions

1. **NFQ and Manibo auth stay separate.** NFQ must have its own OIDC client,
   redirect URIs, and runtime secret wiring. Do not reuse the live Manibo OIDC
   client or treat tenant labels as enough separation.

2. **Temporary public DNS is Route53-backed, not Cloud DNS-backed.**
   `jakitlabs.com` is currently hosted in Route53. Until NFQ gives its own
   domain, the temporary records must be created there after the GCP public
   edge exists.

3. **Public edge must expose both web and API.** The temporary contract is:
   - `nfq.jakitlabs.com` for the web app
   - `api.nfq.jakitlabs.com` for the API

4. **Do not create DNS before the target exists.** First create the GCP public
   load-balancer/ingress target, then create Route53 records pointing at the
   real public address or hostname.

5. **Verification is part of the milestone, not an afterthought.** The work is
   only complete when login, migrations, runtime health, and public routing are
   all proven against the live production stack.
