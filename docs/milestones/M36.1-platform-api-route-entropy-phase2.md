## Goal

Reduce route-layer entropy inside the grouped `platform_api.routes` packages by
turning the route layer into an explicit contract instead of a loose convention.
M36.1 is still deletion-first and behavior-preserving, but its first job is to
define and enforce what code is allowed to live in `routes/`, what must move to
package-local support seams, and which oversized route files must be split so
future agents do not recreate flat sprawl under new names.

## Design Decisions

1. **`apps/api` stays a thin shell** - route packages own HTTP wiring only.
   Code that can run without FastAPI should live below `apps/api`, not inside
   route files.
2. **Behavior-preserving only** - endpoint paths, tags, auth contracts,
   response models, and mounted router behavior stay unchanged.
3. **Route package contract first** - root `routes/` may contain only domain
   packages, `health.py`, and explicitly allowlisted aggregators or temporary
   compatibility shims. Root-level support modules are technical debt, not a
   normal pattern.
4. **Named package-local seams only** - route support code must live in
   explicit package-local owners such as `schemas.py`, `dependencies.py`,
   `presenters.py`, `streams.py`, or `runtime.py`. Generic dumping grounds like
   `utils.py`, `helpers.py`, `support.py`, or `*_support.py` are banned in the
   route layer unless explicitly approved.
5. **Delete before extract** - remove dead compatibility seams and duplicated
   private helpers before introducing any new helper module.
6. **500 LOC hard target** - selected hotspot route files must land under 500
   LOC by the end of the milestone. Anything above that remains structural debt.
7. **Delete lines, not just shuffle lines** - the target is a net route-layer
   reduction of roughly 1.2k-2.0k LOC excluding the later observability split.
8. **Observability is out of scope** - `routes/observability/router.py` remains
   a dedicated follow-on milestone because its 8k LOC size makes it a different
   risk class.
9. **Inventory proof gates deletions** - a shim can only be removed after caller
   evidence shows no live imports remain.
10. **Guards land with the cleanup** - the milestone ends by adding architecture
    checks that stop route-root support sprawl and hotspot regrowth.

## Non-Goals

- Splitting `apps/api/src/platform_api/routes/observability/router.py`
- Changing endpoint paths, tags, auth rules, or response contracts
- Removing the M36 package topology or compatibility strategy wholesale
- Creating a generic route-helper framework across unrelated packages
- Treating route-root support modules as an acceptable steady-state pattern
