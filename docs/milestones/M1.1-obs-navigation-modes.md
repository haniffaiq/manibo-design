## Goal

Split the observability workspace into three navigation modes — Queue, Case, Compare — with omni-channel case queue that treats voice, chat (widget/WhatsApp), email, workflow, incident, runtime, and composition cases in one unified surface. Live cases pin to top with streaming badges.

## Design Decisions

1. **URL-driven mode selection** — no client-side mode state. Queue = `/observability`, Case = `/observability/sessions/[id]`, Compare = query param `?compare=...`
2. **Queue mode is full-width** — no split-screen. Click a case to navigate to case mode.
3. **Case mode collapses queue to breadcrumb back-nav** — evidence rail gets the dominant width
4. **Channel is a badge on the case, not a tab** — operators scan one queue for all channels
5. **Live cases pin to top** — `[Live]` badge, live duration counter, thin separator between live and historical
6. **Channel filter in queue** — dropdown filter for Voice / Web widget / WhatsApp / Email / All, alongside existing case type and status filters
