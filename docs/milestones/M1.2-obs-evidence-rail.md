## Goal

Redesign the evidence rail with severity-based left-border accents, inline gap markers for missing evidence, channel-aware metrics strip, and compact right rail. The evidence rail is the same structure regardless of channel — what changes is which evidence types appear.

## Design Decisions

1. **Left-border severity accents** — neutral: no accent (`│`), warning: amber 3px left border (`▎`), error: red 3px left border (`▎`). Not full-card coloring.
2. **Inline gap markers** — amber dashed rows inserted in the timeline where evidence is missing (e.g., `⚠ Recording expected but asset missing`). Per V2 contract: missing evidence is a contract failure, not a soft badge.
3. **Channel-aware metrics strip** — horizontal bar under case header. Content varies by channel:
   - Voice: V2V, 1st speech, STT, LLM, TTS, Tool wait
   - Chat: 1st response, LLM, messages, lead status
   - Email: delivery, open, click, bounce
   - Workflow: duration, steps, failures
4. **Compact event rows** — tighter padding, kind badge + timestamp on one line, label below. Duration right-aligned.
5. **Right rail is tight** — simple section headers + lists, no cards-within-cards. Sections: Case record, What to do next, Integrity gaps, Related records.
6. **Same rail structure for all channels** — channel determines which evidence types appear, not which UI renders.

## Channel-Specific Evidence Map

| Element | Voice | Web Chat | WhatsApp | Email |
|---------|-------|----------|----------|-------|
| Metrics strip | V2V, STT, LLM, TTS, tool wait | 1st response, LLM, messages, lead status | 1st response, delivery, messages | delivery, open, click, bounce |
| Transcript | Audio-synced STT turns | Message-by-message chat | Message-by-message chat | Template content |
| Media | Audio player | — | — | — |
| Delivery | — | — | Sent/delivered/read receipts | SMTP/open/click tracking |
| Lead capture | Post-call extraction | Inline capture during chat | Inline capture | Linked from source session |
| Gap markers | Recording missing | Session dropped | Delivery failed/retrying | Bounce/suppressed |
| Right rail extras | DID, SIP, voice stack | Widget, locale, page URL | Account, thread, phone | Template, sequence, guardrails |

## Component Foundation

Adopt these components into `packages/ui/src/components/` (copy source from shadcn, adapt to our design tokens — same pattern as existing Modal wrapping `@radix-ui/react-dialog`):

**Tier 1 (adopt in M1.2):**
- **Resizable** (`react-resizable-panels`) — drag-to-resize queue/detail split and compare view panels
- **ToggleGroup** (`@radix-ui/react-toggle-group`) — timeline filter chips with proper selection state
- **Accordion** (`@radix-ui/react-accordion`) — collapsible evidence sections (replaces raw `<details>`)
- **Sheet** (`@radix-ui/react-dialog`, already installed) — slide-out event inspector panel
- **Skeleton** (pure CSS) — shimmer loading states for cards, timeline, metrics

**Audio waveform timeline** (inspired by LiveKit Telephony Console):
- Waveform visualization synced to evidence rail timestamps
- Scrub through audio and see transcript + events pinned to same position
- Agent vs caller waveforms in different colors
- Event markers on the waveform (tool calls, route decisions, interruptions)

**Tier 2 (adopt in M1.2 if time permits, otherwise defer):**
- **DatePicker** (`react-day-picker`) — replace raw `<input type="date">` with calendar + preset ranges
- **Popover** (`@radix-ui/react-popover`) — richer filter dropdowns with search
- **ScrollArea** (`@radix-ui/react-scroll-area`) — consistent scrollbar styling

**Do NOT install shadcn CLI.** Copy component source and adapt to our `var(--color-*)` token system.

- No evidence source attribution tags (backend doesn't support it)
- No visual connection lines or causality graphs
- No live streaming (that's M1.3)
- No new backend APIs — uses existing `ObservabilityRunDetailResponse` fields
