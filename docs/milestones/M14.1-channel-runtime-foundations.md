## Goal

Create the shared Layer 2 channel-runtime contract that sits beside connectors instead of inside them. This milestone defines provider account/runtime state, runtime lifecycle, routing scope, delivery/retry state, and operator-visible status needed by managed interactive channels. It is the planning-only foundation for later web-chat runtime operations, WhatsApp interactive runtime, and any future Slack-interactive work if a requirement is added.

## Design Decisions

1. **Channel runtime is separate from connectors** -- connectors answer "which external API do I call now"; channel runtime answers "how is this live channel account operated over time."
2. **Provider account/runtime is tenant-scoped installed channel state** -- auth, webhook registration, health, and lifecycle state do not live in `public.connectors`.
3. **Reuse the typed control-plane direction from M8** -- runtime lifecycle and operator surfaces must align with the control-plane discipline instead of inventing ad hoc status endpoints.
4. **Session/thread ownership is explicit** -- interactive channels must persist routing/thread truth instead of burying it inside provider payloads.
5. **Layer 2 stays generic** -- a future shared channel-runtime package family in `packages/platform-core/` will own the shared contract/state/services; transport-specific behavior arrives through Layer 3 channel packages and provider packs. The exact package path stays TBD until this planning milestone is explicitly activated.
