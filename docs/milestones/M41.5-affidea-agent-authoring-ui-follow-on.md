## Goal

Implement the future authoring model from design-doc section 8.7 only after
the first Affidea voice-booking agent is working through file-backed and
governed agent-definition paths. This milestone turns the typed
`source_yaml -> compiled_config` model into operator-facing UI and API behavior
without making runtime execute draft YAML.

## Design Decisions

1. This milestone is parked because the design doc explicitly makes full final
   UI shape a non-goal for the first voice-booking version.
2. The UI edits a typed model. It does not freehand-generate arbitrary form
   controls from YAML keys.
3. `source_yaml` remains the canonical authored artifact; `compiled_config` is
   the resolved runtime artifact.
4. Runtime consumes only compiled config. Draft source YAML is never executed
   directly.
5. This milestone must reuse the governed compiler path proven by M41.3.

## Acceptance Criteria

- [ ] UI state round-trips through a typed model into canonical `source_yaml`.
- [ ] Compile preview shows deterministic `compiled_config` without publishing
      a runtime artifact.
- [ ] Runtime execution remains blocked for drafts and uses only compiled,
      published config.
- [ ] Prompt, KB, workflow, handoff, STT, and capture-policy edits are explicit
      typed controls rather than arbitrary YAML-derived forms.
- [ ] Existing Affidea runtime profile from M41.3 remains publishable without
      requiring the UI path.
