## Goal

Cut the merge-gate heavy job wall-clock by parallelising the in-cluster E2E phase and the Playwright web E2E suite inside the existing per-job k3d isolation model. Today the 20-minute in-cluster pytest phase and the Playwright run execute single-threaded despite the cluster, runner, and test fixtures already being isolation-safe enough to run wider. Keeping the M26.4 per-job-cluster contract intact, we introduce `pytest-xdist` with a conservative worker count and lift Playwright off `workers: 1` in CI, measured against a green streak before widening.

- `Release-control CI provides deterministic PR proof, runner-lane routing, and merge finalization for client delivery changes`

## Current Status Snapshot

1. `tools/scripts/e2e/run-k3d-e2e.sh:665` runs `uv run python -m pytest "$@" -q --tb=short` with no `-n` flag. Wall-clock on merge-gate main push is ~20m; queue to first-job-start is <60s (evidence: 30 recent merge-gate runs, 2026-04-15 query).
2. Database isolation is already xdist-safe. `packages/platform-core/tests/e2e/conftest.py:309-315` scopes `public_pool` to function, and `tenant_ctx` (line 400-464) provisions a fresh `tenant_id`, `tenant_schema`, `issuer` per test, so per-worker DB provisioning is not required.
3. The one autouse session-scope fixture that binds a host port (`_native_livekit_server`, `packages/platform-core/tests/e2e/conftest.py:183-238`; `packages/grove-voice-livekit/tests/e2e/conftest.py:43-54`) short-circuits on Linux / CI where `_native_livekit_requested()` / `_is_macos()` is false. The fixture yields without starting a process. CI parallelism is not blocked by port 7880 contention.
4. Workflow IDs that tests submit to Temporal are derived from per-test `call_id`/`tenant_ctx` UUIDs, so the shared Temporal worker processing `grove-agent` can still be used by multiple xdist workers without workflow-ID collision. The existing session-scope `grove_task_queue` fixture (`conftest.py:304-306`) does not need to change for the conservative rollout.
5. `apps/web/playwright.config.ts:22` hardcodes `workers: process.env.CI ? 1 : undefined`. `fullyParallel: true` is already set (line 19). Auth is per-page-context via `primeSessionCookie` (`apps/web/e2e/session-helpers.ts:17-44`), no shared `storageState`, no `globalSetup`/`globalTeardown`. Widening CI workers is a config change, not a test refactor.
6. `pytest-xdist` is not in dev dependencies today (`pyproject.toml:11-15`). Must be added.

## Design Decisions

1. **Keep per-job k3d isolation exactly as-is.** M26.4's contract (`K3D_RESET_CLUSTER_ON_CI=true`, per-job cluster names, per-job port slots) is not touched. All parallelism happens inside a single cluster owned by a single GitHub Actions job.
2. **Opt-in, env-gated rollout.** `run-k3d-e2e.sh` reads `E2E_PYTEST_WORKERS` (integer or `auto`). Default is `1` to keep current behaviour. CI sets `E2E_PYTEST_WORKERS=2` initially. We grow to `4` / `auto` only after three consecutive green merge-gate main runs at the previous step.
3. **Conservative worker count, not `auto`, on day one.** `-n 2` halves wall-clock if isolation holds. `-n auto` on a 4-core Hetzner heavy runner would spawn more workers than cores for useful work; benchmark first.
4. **Playwright CI widening is a config change with an env escape hatch.** Set `workers: process.env.CI ? Number(process.env.PLAYWRIGHT_WORKERS ?? 4) : undefined`. If a spec reveals cross-worker flake, revert via env without a code change.
5. **No test-code refactors in this milestone.** If xdist surfaces a cross-test state leak, the fix belongs in a follow-on task under the test's owning milestone, not here. M26.8 lands the harness and the first safe step-up; it does not chase rotten fixtures.
6. **Rollback path is a one-line env flip.** No new infra, no new runner roles, no workflow topology changes. If merge-gate goes red, flip `E2E_PYTEST_WORKERS=1` / `PLAYWRIGHT_WORKERS=1` in the workflow env and move on.

## Non-Goals

- Not migrating to vCluster or any warm/shared cluster scheme. See `wiki/queries/2026-04-15-ci-speed-vcluster-vs-parallelism.md` for why.
- Not splitting `admission-product` into parallel GitHub Actions jobs (that work is M26.9, gated on runner-capacity decision).
- Not fixing the broken nightly regression-e2e workflow. That is M26.10.
- Not rewriting any session-scope fixture that is currently a no-op on CI Linux (`_native_livekit_server`). We leave macOS real-provider fixtures alone.
- Not adding per-worker Temporal task queue namespacing on day one. If a workflow-ID collision shows up in T02 rollout, file a follow-on task; do not block this milestone.
- Not changing `K3D_RESET_CLUSTER_ON_CI`, `configure-k3d-ci-env.sh`, or any M26.4 isolation contract.
- Not adding pytest-xdist to the `packages/grove/tests/unit` architecture tree. Scope is the in-cluster E2E phase driven by `run-k3d-e2e.sh` plus the Playwright web lane.
