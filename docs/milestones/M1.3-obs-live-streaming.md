## Goal

Make live cases stream evidence in real-time with channel-appropriate operator actions. Voice cases stream transcript + events via SSE with audio listen-in. Chat cases stream messages. The evidence rail becomes a live feed that transitions to a forensic record when the interaction ends.

## Design Decisions

1. **Same evidence rail, streaming mode** — live cases use the same `EvidenceRail` component. New events append at the bottom via SSE. No separate "live dashboard."
2. **Operator action bar** — pinned at top of case view, above metrics strip. Actions vary by channel:
   - Voice: Listen in, Take over call, Transfer now
   - Chat (widget/WhatsApp): Watch session, Join as operator, Escalate to human
   - Email: no live actions (async channel)
   - Workflow: no live actions (background)
3. **Live phase indicator** — right rail section showing what the interaction is doing right now (agent speaking, guest typing, tool running, idle).
4. **Graceful transition** — when interaction ends, operator actions disappear, live badges drop, metrics finalize, and the case becomes a historical record. No page reload.
5. **Reuse existing SSE infrastructure** — `useSseStream` hook at `apps/web/src/hooks/use-sse-stream.ts` handles reconnection, backoff, event parsing. `support-drawer.tsx` demonstrates dual-stream pattern (transcript + ops). Reuse both.
6. **LiveKit integration for voice listen-in** — `POST /calls/{call_id}/livekit-token` returns subscribe-only token (15min TTL). `POST /calls/{call_id}/livekit-operator-token` returns publish-enabled token for takeover. `livekit-client@2.17.2` already installed.
7. **Auto-scroll with user override** — live evidence rail auto-scrolls to newest event. If the user scrolls up to inspect an earlier event, auto-scroll pauses until they scroll back to bottom.

## Critical Implementation Notes

1. **T01 reuses existing call-ops action handlers.** `support-drawer.tsx` already implements Take Over and Transfer via `platformApiRequest()`. Extract the action dispatch logic, don't rebuild it.

2. **T02 dual-stream pattern.** Two `useSseStream` calls running concurrently — one for `/transcript/stream`, one for `/ops/stream`. Both track `after_seq` in a ref. Merge results into a single chronologically-ordered feed. The `support-drawer.tsx:93-118` pattern is the template.

3. **T03 mapper is critical.** SSE transcript events (`{seq, speaker, timestamp, text}`) and ops events (`{seq, event_type, occurred_at_ms, summary, payload}`) must transform into `ObservabilityTimelineItem` shape (`{id, kind, label, detail, severity, occurred_at, ...}`). Without this, SSE events can't render in the evidence rail.

4. **T04 auto-scroll.** Use a "stick to bottom" pattern: track whether the user is scrolled to the bottom. If yes, `scrollToItem(items.length - 1)` on every new event. If the user scrolls up, stop auto-scrolling. Resume when they scroll back to bottom. The react-window `List` ref has `scrollToItem()`.

5. **T06 LiveKit connection.** Use `livekit-client` Room class: `new Room() -> room.connect(url, token) -> room.on('trackSubscribed', ...)`. The subscribe-only token from `/livekit-token` prevents the operator from broadcasting. Disconnect on unmount or case transition.

6. **T08 transition detection.** When SSE sends `event: end`, the case has ended. Alternatively, poll `detail.data?.summary.status` — when it changes from Running to Completed/Failed, trigger transition. Remove OperatorActionBar, stop SSE streams, disconnect LiveKit room, refresh detail data to get final metrics.

7. **T09 duration counter.** Already have `isRunningStatus()` and the LIVE badge from M1.2. Add elapsed time: `Date.now() - new Date(run.started_at).getTime()`, update via `setInterval(1000)`. Display as `formatDurationMs()`. Clear interval on unmount.

8. **T12 metrics placeholder.** When `isRunningStatus(detailSummary.status)` is true, the metrics strip should show metric labels with "--" values instead of hiding. When the case transitions to historical, refresh `detail` to get final metrics.
