## Goal

Improve the frontend quality of all calls surfaces without backend changes:

- **Admin calls list** — `/admin/calls` (deployment console)
- **Admin call replay** — `/admin/calls/:callId` (deployment console)
- **Tenant call-ops dashboard** — `/call-ops` (tenant console)
- **Tenant call history** — `/call-ops/history` (tenant console)
- **Tenant alerts** — `/call-ops/alerts` (tenant console)

Replace dummy/mock actions with honest frontend patterns, extract inline components, improve loading/empty states, add pagination, and polish responsive behavior.

## Current Layout and Styling Reference

### Admin Calls List (`/admin/calls`)

**Layout:**
- Full-height flex column (`h-[calc(100vh-3.5rem)]`)
- Three vertical bands: breadcrumb bar → sub-header (tabs + tenant filter) → scrollable content
- Tab switcher: `Live` / `History` in a segmented control (`rounded-lg border bg-[var(--color-bg-subtle)] p-0.5`)
- Tenant filter: custom inline `TenantDropdown` with click-outside and keyboard handling

**Live tab:**
- Search input + state filter pills (`all`, `in_progress`, `on_hold`, `supervised`, `ringing`)
- Full-width data table: columns `AGENT | TENANT | CALLER | INTENT | DURATION | STATUS | COST`
- Table header: `bg-[var(--color-bg-subtle)]`, text-[11px] uppercase tracking-wider
- Table rows: clickable, `hover:bg-[var(--color-bg-subtle)]`, `border-b`
- Live detail view: caller header card + 3-column action grid (`grid grid-cols-3 gap-3`) for Listen/Takeover/End + transcript + context sidebar (`lg:grid-cols-[1fr_340px]`)

**History tab:**
- Search input + outcome dropdown + CSV export button
- Full-width data table: columns `STARTED | TENANT | AGENT | CALLER | INTENT | DURATION | OUTCOME | COST | TOOLS`
- History detail: caller header card + action buttons + transcript preview + context sidebar (`lg:grid-cols-[1fr_340px]`)

**Dummy actions present:** `dummyNotice()`, `downloadDummyFile()`, `exportCallsCsv()`, `exportTranscriptTxt()`, `downloadDummyRecording()` — all use DOM manipulation or hardcoded fake data.

### Admin Call Replay (`/admin/calls/:callId`)

**Layout:**
- Full-height flex column (`h-[calc(100vh-3.5rem)]`)
- Breadcrumb → scrollable content with `space-y-5`

**Dummy actions present:** `dummyNotice()`, `downloadDummyFile()` — same pattern as admin calls.

- `PageFrame` wrapper with `width="workspace"` and `px-6 py-8`

**Styling:** Most mature — uses design system tokens, `PageFrame`/`PageHeader`, `Badge`, `Button`, `Card`, `StatusMessage`. No dummy actions. Real API wiring through `getCallOpsDashboardData`, `mintLiveKitToken`, `enrichCallsWithEscalation`.

### Tenant Call History (`/call-ops/history`)

**Layout:**
- `PageFrame` with `px-6 py-8`
- Vertical flow: `PageHeader` → filters `Card` → master-detail split
- Filters: `md:grid-cols-2` — Driver ID, Phone, Outcome (select), Started After, Started Before (datetime-local)
- Master-detail: `lg:grid-cols-[minmax(0,2fr)_minmax(0,3fr)]`

### Tenant Alerts (`/call-ops/alerts`)

**Layout:**
- `PageFrame` with `px-6 py-8`
- Vertical flow: `PageHeader` → filters `Card` → alert queue
- Filters: `md:grid-cols-3` — severity (select), status (select), since (datetime-local)
- Alert cards: `space-y-2`, each card is `flex items-start gap-4 rounded-lg border border-l-4`
  - Left border accent by severity: info=neutral-400, warning=warning-500, critical=error-500
  - Content: event type + severity badge + status badge, message, timestamp + entity reference
  - Actions: Acknowledge (outline) + Resolve (destructive) buttons
- Auto-refresh: SWR `refreshInterval: 10_000` but no visual indicator

## Design Decisions

1. **Frontend-only.** No new backend endpoints, no API contract changes. Where the current frontend uses mock/dummy functions, replace with honest "not yet available" states or wire to existing API clients.
2. **No new design system components** unless a pattern repeats 3+ times across these pages. Prefer composing existing `@grove/ui` primitives.
3. **Admin calls page** keeps the live/history tab structure. Improvements focus on replacing dummy actions and extracting the inline TenantDropdown.
4. **Call replay page** keeps the timeline/transcript/inspector layout. Improvements focus on the non-functional audio player and dummy download actions.
5. **Tenant call-ops pages** are the most mature. Improvements are limited to history pagination, alerts UX, and responsive consistency.
6. **Preserve existing color tokens and typography.** No new design language — follow the existing `var(--color-*)` + Tailwind utilities pattern.
