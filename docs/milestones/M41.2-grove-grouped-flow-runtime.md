## Goal

Add a product-agnostic Grove runtime capability for grouped flow authoring that
can compile parent graphs with specialist subgraphs. This lets Affidea model
front desk, consultation, MRI, and auth as real workflow groups without
copying LiveKit-agent handoff mechanics into the platform.

## Design Decisions

1. This milestone is generic Grove work. It must not mention Affidea, clinics,
   patients, MRI, or any Layer 3 business noun in `packages/grove/**`.
2. Current flat `flow_definition` remains supported and unchanged.
3. Grouped flow authoring compiles to LangGraph parent/subgraph structure.
   Subgraphs use shared Grove state and namespaced `flow_context` keys for v1;
   arbitrary custom TypedDict private state is deferred until there is a second
   real consumer.
4. The parent graph owns durable checkpointing. Specialist subgraphs inherit
   the parent checkpointer by default, matching current LangGraph guidance.
5. `Command`/routing semantics stay explicit and typed through Grove's existing
   route-decision model; no opaque multi-agent handoff convention is added.
6. Checkpoint, prompt-cache, and deterministic tool-order behavior must be
   covered by tests before this is usable by a voice agent.

## Acceptance Criteria

- [ ] Existing flat `flow_definition` tests pass without behavior changes.
- [ ] A grouped-flow fixture can model a parent router with two specialist
      groups and one auth/resume group.
- [ ] Parent graph checkpointing resumes through grouped subgraph nodes across
      multiple calls using a stable `thread_id`.
- [ ] Checkpoint history exposes enough state to replay or fork a grouped-flow
      turn for debugging without adding subgraph-local persistence.
- [ ] Subgraph node spans include group and node names without business data.
- [ ] Tool lists remain deterministic in generated model payloads.
- [ ] `packages/grove/**` keeps architecture-boundary tests green and imports
      no platform or solution code.
- [ ] Docs clearly explain when to use grouped flow authoring versus current
      flat flow definitions.

## Non-Goals

- No Affidea-specific implementation.
- No autonomous agent runtime work from M33.
- No LangGraph Platform server adoption.
- No generic vector retrieval or memory system.
- No custom per-subgraph state schema DSL beyond namespaced `flow_context`.
