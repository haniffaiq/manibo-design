## Goal

Prove that the Affidea voice-booking agent is observable, testable, and ready
for controlled NFQ production rollout. This milestone turns the local runtime
integration into evidence: synthetic and real-provider evals, k3d proof, GCP
runtime proof, alert coverage, docs, and a rollout checklist.

## Design Decisions

1. A production-readiness claim requires runtime evidence, not only unit tests.
2. Voice proof uses the shared artifact envelope and marker semantics described
   by the voice/testing docs.
3. Required spans/events follow the design doc: intake, FAQ, consultation,
   MRI, auth, slot search, slot lock, booking completion, and handoff.
4. Provider failure metrics and alert routes build on M39. If M39 is not done,
   this milestone stays blocked for production readiness.
5. The first rollout is controlled and NFQ-scoped. It must not change default
   agents for non-NFQ tenants.

## Acceptance Criteria

- [ ] Synthetic evals cover consultation happy path, MRI happy path, auth resume,
      unsafe MRI handoff, explicit human request, and unavailable slot recovery.
- [ ] Voice artifacts include transcript, runtime events, graph route events,
      tool calls, latency evidence, and final booking/handoff outcome.
- [ ] Required spans and metrics carry `call_id`, `tenant_id`, `current_flow`,
      `originating_flow`, provider, service ID when known, and slot ID when
      known, with sensitive patient data redacted.
- [ ] k3d proof shows the browser voice flow starts, runs through at least one
      booking scenario, and persists post-call artifacts.
- [ ] GCP/NFQ proof shows the published Affidea agent can be resolved by the
      live runtime and that provider-failure alert coverage is active.
- [ ] Documentation and runbooks name the exact rollout steps, rollback steps,
      known limitations, and eval artifact paths.
- [ ] Eval output makes an explicit deterministic-KB decision: keep M41.6
      parked, or activate it with concrete failed scenarios and expected lift.
- [ ] Existing clinic-registration and non-NFQ tenants are unaffected.
