# AI Worker Platform — Requirements Checklist

> **Status: legacy reference, not actively maintained.** This checklist is kept as a historical scope/requirements snapshot. Specific route names, file paths, spec filenames, and `live` claims here may not match the current codebase — surfaces have been added, renamed, or removed since this was last touched. Do **not** treat this document as the current source of truth and do **not** update it as part of unrelated PRs. For current state, consult the wiki and `docs/arch/generated/api_inventory.md`.

**Sources:** `docs/requirements/vox.md` (VOX SOW, DRAFT 0.1) · `docs/requirements/nfq.md` (NFQ Platform Vision v1.1)
**Purpose:** Delivery checklist — track what the platform must provide for each client to go live.
**Columns:** Requirement · User Story · Client · UI · API · Repo · Tests · Wave · Check · Prove
**Client:** `Both` = required by both · `VOX` = VOX-Sprachschule only · `NFQ` = NFQ delivery slice (logistics demo tenant: Hoptrans)
**UI:** `✓` = requires a UI screen or component · **API:** `✓` = requires a backend API endpoint or integration
**Repo:** repo implementation status (as of 2026-03-06) — ✅ implemented · ⚠️ partial · 🐛 broken · ❌ missing · — not repo-verifiable
**Wave:** implementation wave home — `7`, `8`, `9`, `External` (and `8+9` when Wave 8 delivers the NFQ logistics slice and Wave 9 generalizes)
**Tests:** `U/I/E` = unit/integration/e2e coverage — ✅ covered · ❌ missing · — not required / external dependency

Repo status is based on **current code in this repository**. It is **not** a production sign-off.
Use **Check/Prove** for production validation evidence (screenshots, tests, logs, demo recordings).
